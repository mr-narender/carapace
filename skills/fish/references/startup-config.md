# Fish Startup and Configuration

In-depth reference for fish's startup process, configuration files, autoloading, XDG paths, environment variables, and where to install completions.

## Startup File Decision Flow

### Interactive Shell

1. **Configuration snippets** (`.fish` files in `conf.d` directories, alphabetical order)
2. **System-wide config** (`$__fish_sysconf_dir/config.fish`, usually `/etc/fish/config.fish`)
3. **User config** (`$__fish_config_dir/config.fish`, usually `~/.config/fish/config.fish`)

### Non-Interactive Shell (script execution)

Same as interactive — all config files are sourced. Use `status --is-interactive` to guard interactive-only settings.

### Login Shell

Fish does **not** distinguish between login and non-login shells for configuration. Use `status --is-login` to detect login shells if needed.

### Key Difference from Bash

Fish reads the same configuration files regardless of whether the shell is interactive or a login shell. There is no separate `.bash_profile` / `.bashrc` split.

## Configuration File Locations

### Directory Variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `$__fish_config_dir` | `$XDG_CONFIG_HOME/fish` (default `~/.config/fish`) | User config dir |
| `$__fish_sysconf_dir` | `/etc/fish` | System config dir |
| `$__fish_user_data_dir` | `$XDG_DATA_HOME/fish` (default `~/.local/share/fish`) | User data dir |

### conf.d Snippet Directories (loaded first, alphabetical order)

| Directory | Purpose |
|-----------|---------|
| `$__fish_config_dir/conf.d/*.fish` | User snippets |
| `$__fish_sysconf_dir/conf.d/*.fish` | System snippets |
| `$__fish_user_data_dir/vendor_conf.d/*.fish` | Third-party snippets |
| `$XDG_DATA_DIRS`/`fish/vendor_conf.d/*.fish` | Vendor snippets (e.g., `/usr/share/fish/vendor_conf.d`) |

**Deduplication**: If multiple files have the same name across directories, only the first is executed.

### config.fish

| File | When Read |
|------|-----------|
| `$__fish_sysconf_dir/config.fish` | After conf.d snippets |
| `$__fish_config_dir/config.fish` | After system config (user overrides) |

### Typical `~/.config/fish/config.fish`

```fish
# Environment
set -gx EDITOR vim
set -gx GOPATH ~/go

# Add to PATH
fish_add_path ~/bin ~/go/bin

# Aliases (as functions)
function ll
    ls -la $argv
end

# Interactive-only settings
if status --is-interactive
    set -g fish_greeting

    # Abbreviations
    abbr -a gco git checkout
    abbr -a gs git status
end
```

## Function Autoloading

Functions are autoloaded from directories in `$fish_function_path`:

| Directory | Purpose |
|-----------|---------|
| `~/.config/fish/functions/` | User functions |
| `/etc/fish/functions/` | System-wide functions |
| `~/.local/share/fish/vendor_functions.d/` | Third-party functions |
| `/usr/share/fish/vendor_functions.d/` | Vendor-shipped functions |
| `/usr/share/fish/functions/` | Fish built-in functions |

### File Naming

Function files must be named `<function-name>.fish`. When a function is first called, fish loads the corresponding file.

### Lazy Loading

Fish autoloads functions on first use — not at startup. This keeps startup fast even with many function files.

## Completion Autoloading

Completions are autoloaded from directories in `$fish_complete_path`:

| Directory | Purpose |
|-----------|---------|
| `~/.config/fish/completions/` | User completions |
| `/etc/fish/completions/` | System-wide completions |
| `~/.local/share/fish/vendor_completions.d/` | Third-party completions |
| `/usr/share/fish/vendor_completions.d/` | Vendor-shipped completions |
| `/usr/share/fish/completions/` | Fish built-in completions |
| `~/.cache/fish/generated_completions/` | Auto-generated from man pages |

### File Naming

Completion files must be named `<command-name>.fish`. When a command is first invoked, fish loads the corresponding completion file.

### Installing Completions

**For a single user:**

```fish
# Drop completion file in:
~/.config/fish/completions/myapp.fish
```

**System-wide:**

```fish
# Drop completion file in:
/etc/fish/completions/myapp.fish
```

**Via package manager (vendor):**

```fish
# Drop completion file in:
/usr/share/fish/vendor_completions.d/myapp.fish
```

**Via carapace:**

Carapace generates completions dynamically. The snippet registers a completion function that calls the application's `_carapace` subcommand:

```fish
# Generated by: myapp _carapace fish | source
complete -e 'myapp'
complete -c 'myapp' -f -a '(_myapp_completion)' -r
```

### Man Page Completions

Fish auto-generates completions from man pages on first use. These are stored in `~/.cache/fish/generated_completions/` and provide basic option completions for commands that don't have dedicated completion scripts.

## PATH Handling

### fish_add_path

The recommended way to add directories to PATH:

```fish
fish_add_path ~/bin           # Prepend to PATH
fish_add_path --append ~/bin  # Append to PATH
fish_add_path --move ~/bin    # Move to front if already present
```

### fish_user_paths

The `$fish_user_paths` variable is a universal list of directories that are automatically prepended to `$PATH`:

```fish
set -U fish_user_paths ~/bin ~/go/bin
```

Changes to `fish_user_paths` take effect immediately in all fish sessions.

### Key Difference from Bash

- Fish uses `fish_add_path` or `set -a PATH` instead of `PATH="$HOME/bin:$PATH"`
- PATH is a list, not a colon-delimited string (fish auto-converts)
- `fish_user_paths` provides a universal, cross-session PATH modification

## Universal Variables

Universal variables are shared across all fish instances and persist to disk.

### Storage

- File: `~/.config/fish/fish_variables`
- Format: internal (not meant to be edited directly)
- Sync: `EnvUniversal::sync()` handles reading and merging
- Notification: filesystem-based notification for cross-process sync
- Atomic writes via `rewrite_via_temporary_file()`

### Common Universal Variables

```fish
set -U fish_color_command blue       # Syntax highlighting color
set -U fish_greeting                 # Disable greeting
set -U fish_user_paths ~/bin        # PATH additions
set -U EDITOR vim                    # Default editor
```

### How Universal Variables Sync

1. When a universal variable is modified, fish writes to `fish_variables`
2. Other fish instances detect the change via filesystem notification
3. `EnvUniversal::sync()` reads and merges changes
4. Changes propagate to all running fish instances

## Environment Variables Set by Fish

| Variable | Description |
|----------|-------------|
| `fish_pid` | Current shell's PID |
| `fish_version` | Fish version string |
| `fish_greeting` | Greeting message (customizable) |
| `EUID` | Effective user ID (set at startup) |
| `HOME` | User's home directory |
| `PWD` | Current working directory |
| `USER` | Current username |
| `SHLVL` | Shell nesting level |

## fish_config

The `fish_config` command provides a web-based and terminal-based configuration interface.

### Theme Management

```fish
fish_config theme show           # Show all themes in terminal
fish_config theme choose lava   # Choose a theme
fish_config theme save           # Save current theme
fish_config theme dump          # Dump current theme as variables
```

### Prompt Management

```fish
fish_config prompt show          # Show all prompts in terminal
fish_config prompt choose disco # Choose a prompt
fish_config prompt save          # Save selection
```

### Web Interface

```fish
fish_config                      # Open web configuration
fish_config colors               # Open colors page
fish_config prompt               # Open prompt page
```

## Command-Line Flags

| Flag | Description |
|------|-------------|
| `--no-config` / `-N` | Don't read configuration files |
| `--init-command CMD` | Run CMD before reading config |
| `--private` / `-P` | Private mode (no history saved) |
| `--login` | Start as login shell |
| `--interactive` | Start as interactive shell |
| `--command CMD` / `-c CMD` | Execute CMD |
| `--debug LEVEL` | Enable debug output (1-128) |
| `--debug-stack-frames N` | Show N stack frames in debug output |
| `--profile FILE` | Profile execution to FILE |

## Event Handlers

Fish supports event handlers triggered by various events:

### Named Events

| Event | When Triggered |
|-------|---------------|
| `fish_prompt` | Before displaying prompt |
| `fish_preexec` | Before executing a command |
| `fish_postexec` | After executing a command |
| `fish_exit` | When shell exits |
| `fish_cancel` | When command line is cancelled |
| `fish_focus_in` | When terminal gains focus |
| `fish_focus_out` | When terminal loses focus |

### Signal Events

```fish
function on_sigusr1 --on-signal SIGUSR1
    echo "Received SIGUSR1"
end
```

### Variable Change Events

```fish
function on_mode_change --on-variable fish_mode
    echo "Mode changed to $fish_mode"
end
```

### Job Exit Events

```fish
function on_job_exit --on-job-exit %self
    echo "Job exited"
end
```

## XDG Base Directory Compliance

Fish follows the XDG Base Directory specification:

| XDG Variable | Fish Default | Fish Variable |
|--------------|-------------|---------------|
| `XDG_CONFIG_HOME` | `~/.config` | `$__fish_config_dir` |
| `XDG_DATA_HOME` | `~/.local/share` | `$__fish_user_data_dir` |
| `XDG_CACHE_HOME` | `~/.cache` | `~/.cache/fish` |
| `XDG_CONFIG_DIRS` | `/etc/xdg` | System config dirs |
| `XDG_DATA_DIRS` | `/usr/share` | System data dirs |

### Directory Layout

```
~/.config/fish/
├── config.fish              # User configuration
├── conf.d/                  # Configuration snippets
│   └── *.fish
├── completions/             # User completions
│   └── <command>.fish
├── functions/               # User functions
│   └── <function>.fish
└── fish_variables           # Universal variables storage

~/.local/share/fish/
├── vendor_completions.d/    # Third-party completions
├── vendor_functions.d/      # Third-party functions
├── vendor_conf.d/           # Third-party config snippets
└── fish_history             # Command history

~/.cache/fish/
└── generated_completions/   # Auto-generated from man pages
```

## Common Patterns

### Adding a Directory to PATH

```fish
# In config.fish (session-only)
fish_add_path ~/bin

# Universal (persists across sessions)
set -U fish_user_paths ~/bin $fish_user_paths
```

### Conditional Configuration

```fish
# Interactive-only settings
if status --is-interactive
    abbr -a gco git checkout
    set -g fish_greeting
end

# Login-only settings
if status --is-login
    fish_add_path ~/linux/bin
end
```

### Running Commands on Exit

```fish
function on_exit --on-event fish_exit
    echo fish is now exiting
end
```

### Setting Login Shell

```fish
# Add fish to accepted shells
echo /usr/bin/fish | sudo tee -a /etc/shells

# Change default shell
chsh -s /usr/bin/fish
```

**Warning**: Setting fish as login shell may cause issues on some systems that require Bourne-compatible shells with `/etc/profile` configuration.

## Pitfalls

1. **No login/non-login split**: Unlike bash, fish reads the same config files regardless. Use `status --is-interactive` and `status --is-login` for conditional behavior.
2. **conf.d deduplication**: If multiple files have the same name across directories, only the first is executed. Be careful with naming.
3. **Universal variable race conditions**: Multiple fish instances writing universal variables simultaneously may cause conflicts. Fish uses atomic writes to mitigate this.
4. **PATH ordering**: `fish_add_path` prepends by default. Use `--append` to append instead.
5. **Completion autoloading timing**: Completions are loaded on first use, not at startup. This means `complete -c foo` won't show completions until `foo` has been invoked at least once.
6. **SSH sessions**: SSH typically invokes login shells. If fish is the login shell, it reads the same config files as an interactive shell.

## Comparison with Bash's Startup Model

| Feature | Fish | Bash |
|---------|------|------|
| Login vs non-login | No distinction for config | Different files (`.bash_profile` vs `.bashrc`) |
| Interactive vs non-interactive | Same files, use `status --is-interactive` | `.bashrc` only for interactive |
| conf.d snippets | Yes (alphabetical) | No built-in support |
| Universal variables | Yes (cross-session, file-backed) | No equivalent |
| PATH modification | `fish_add_path`, `fish_user_paths` | `PATH="$HOME/bin:$PATH"` in `.bashrc` |
| Completion installation | Drop in `completions/` directory | Source in `.bashrc` or use `bash-completion` |
| Config file format | Fish script | Bash script |
| Vendor directories | Yes (XDG-based) | No built-in support |
