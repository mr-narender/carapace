# Carapace Library: Fish Shell Integration Deep Dive

Reference for [carapace](https://github.com/carapace-sh/carapace)'s fish completion integration — how the snippet works, how completion output is formatted, and how carapace handles fish-specific edge cases. For cross-shell comparisons, see the **references/shell.md**.

## Source Files

| File | Purpose |
|------|---------|
| `internal/shell/fish/snippet.go` | Fish completion script generation |
| `internal/shell/fish/action.go` | Value formatting (tab-separated `value\tdescription`) |
| `internal/shell/shell.go` | Shared dispatch — message integration, nospace propagation, filtering |
| `complete.go` | Entry point — no fish-specific patching |

## Shell Background

### The `complete` Built-in

Fish registers completion functions via `complete -c command_name -f -a '(completion_function)'`. When the user presses TAB, fish:

1. Identifies the command being completed
2. Invokes the registered completion function (or command substitution in `-a`)
3. Parses the output as newline-separated candidates
4. Each line may contain a **tab character** — text before the tab is the completion value, text after is the description
5. Displays candidates in the pager with descriptions

### The `commandline` Built-in

Key options used in carapace's snippet:

| Option | Meaning |
|--------|---------|
| `-c` / `--cut-at-cursor` | Print command line only up to cursor position |
| `-p` / `--current-process` | Limit to current process (stops at pipes, `&&`, `;`) |

Combined `commandline -cp` returns the command line up to the cursor, limited to the current process.

### Fish `complete` Flags Used by Carapace

| Flag | Meaning |
|------|---------|
| `-c 'cmd'` | Register completion for command `cmd` |
| `-e` | Erase existing completion (prevents conflicts with built-in fish completions) |
| `-f` / `--no-files` | Disable file completion |
| `-a '(...)'` | Command substitution that produces completion candidates |
| `-r` / `--require-parameter` | Completion requires a parameter |

### Fish Completion Output Format

```
value1\tdescription1
value2\tdescription2
value3\tdescription3
```

- Each line is one completion candidate
- Tab character (`\t`) separates the **value** from the **description**
- If there is no tab, the entire line is the value with no description

### Fish Word Splitting

Fish does **not** split on `IFS` by default. Fish splits command substitution output only on **newlines**, not spaces. This means fish never has the COMP_WORDBREAKS problem that bash has — `origin/main`, `--option=value`, and `user@host` stay as single tokens.

## The Fish Snippet

Generated by `fish.Snippet(cmd)` in `internal/shell/fish/snippet.go`:

```fish
function _example_completion
  set --local data
  IFS='' set data (echo (commandline -cp)'' | sed "s/ $/ ''/" | xargs example _carapace fish 2>/dev/null)
  if [ $status -eq 1 ]
    IFS='' set data (echo (commandline -cp)"'" | sed "s/ $/ ''/" | xargs example _carapace fish 2>/dev/null)
    if [ $status -eq 1 ]
      IFS='' set data (echo (commandline -cp)'"' | sed "s/ $/ ''/" | xargs example _carapace fish 2>/dev/null)
    end
  end
  echo $data
end

complete -e 'example'
complete -c 'example' -f -a '(_example_completion)' -r
```

### Snippet Walkthrough

**1. Define the completion function**

```fish
function _example_completion
  set --local data
```

**2. Open-quote retry logic**

The three-stage retry handles the **open-quote problem**:

1. `''` — append empty single-quoted string (handles normal case and open single quote)
2. `"'"` — close with single quote then double quote (handles `'partial` case)
3. `'"'` — close with double quote (handles `"partial` case)

Each attempt pipes through `sed "s/ $/ ''/"` which appends `''` to lines ending with a space, then through `xargs` to split into arguments.

**3. Output the data**

```fish
echo $data
```

The completion data (tab-separated `value\tdescription` lines) is printed to stdout.

**4. Register the completion**

```fish
complete -e 'example'
complete -c 'example' -f -a '(_example_completion)' -r
```

- `complete -e` erases any existing completion for the command
- `-f` / `--no-files` disables fish's default file completion
- `-a '(_example_completion)'` — the function is invoked as a command substitution
- `-r` / `--require-parameter` — enables completion after space

## No Fish-Side Patching

Fish has no `Patch()` function. The `complete.go` dispatch has no `case "fish"` branch — fish arguments are passed directly to `traverse()` unmodified.

Fish's tokenizer (using the `TOK_ACCEPT_UNFINISHED` flag) is designed for completion — it can handle partially typed, syntactically invalid command lines. The `commandline -cp` output is already a clean representation of the command line up to the cursor.

The open-quote problem is still present but is handled entirely in the snippet (the 3-stage xargs retry), not in Go-side patching.

## Value Formatting: `ActionRawValues()`

`fish.ActionRawValues(currentWord, meta, values)` in `internal/shell/fish/action.go` is the simplest formatter in carapace — only 6 lines of logic.

### Implementation

```go
var sanitizer = strings.NewReplacer(
    "\n", ``,
    "\r", ``,
    "\t", ``,
)

func ActionRawValues(currentWord string, meta common.Meta, values common.RawValues) string {
    vals := make([]string, len(values))
    for index, val := range values {
        vals[index] = fmt.Sprintf("%v\t%v", sanitizer.Replace(val.Value), sanitizer.Replace(val.TrimmedDescription()))
    }
    return strings.Join(vals, "\n")
}
```

### Step 1: Sanitize

The sanitizer strips control characters that would break the output format:

- `\n` → empty — newlines are the record delimiter
- `\r` → empty — carriage returns would corrupt the output
- `\t` → empty — tabs are the field delimiter (value vs description)

### Step 2: Format as Tab-Separated

Each `RawValue` becomes `value\tdescription`. Empty descriptions produce `value\t`.

### Step 3: Join with Newlines

All candidates are joined with `\n`. Fish splits on newlines to get individual candidates.

## Nospace Handling: A Known Limitation

Carapace's `SuffixMatcher` (accessed via `meta.Nospace`) controls whether a trailing space should be added after a completion. **Fish's `ActionRawValues` ignores `meta.Nospace` entirely.** The function receives `meta` as a parameter but never references `meta.Nospace`. There is no mechanism in the tab-separated format to communicate nospace information.

This means **every completion in fish gets a trailing space**. This is wrong for values where nospace was explicitly requested (like directories ending in `/`, or comma-separated values ending in `,`).

In `shell.go`, the shared dispatch adds `*` to the nospace set when messages exist, but since fish's `ActionRawValues` never reads `meta.Nospace`, this has **no effect** for fish completions — it is effectively dead code for the fish path.

## Message Handling

Fish falls into the **`default`** case in `shell.go`'s message handling:

```go
switch shell {
case "elvish", "export", "zsh": // shells with support for showing messages
default:
    values = meta.Messages.Integrate(values, value)
}
```

Messages are **integrated as synthetic completion candidates** via `meta.Messages.Integrate(values, value)`. This method converts messages into fake completion values:

- Display text: `ERR`, `ERR1`, `ERR2`, etc. (incrementing to avoid collisions)
- Value text: `<currentWord prefix>ERR`, `<currentWord prefix>ERR1`, etc.
- Description: the actual error message text
- Style: `style.Carapace.Error`

## Description Display and the Fish Pager

When a completion candidate includes a tab character, fish splits on that tab. The first part becomes the **completion value** (what gets inserted), and the second part becomes the **description** (shown in the pager next to the value).

Fish's pager displays descriptions in a column to the right of the values. Carapace uses `val.TrimmedDescription()` rather than `val.Description` — truncating descriptions that exceed `CARAPACE_DESCRIPTION_LENGTH` (default 80 characters).

Fish's formatter does **not** output any style information. The `val.Style` field from `RawValue` is completely ignored. Fish does support pager color customization via `fish_pager_color_*` variables, but these apply globally, not per-candidate.

## Edge Cases and Known Issues

### 1. Open-Quote Completion

The snippet's 3-stage retry handles the most common open-quote scenarios. **Limitation**: The retry logic is a heuristic that may fail for nested quotes or mixed quote scenarios.

### 2. Trailing Spaces in Arguments

The `sed "s/ $/ ''/"` command handles the edge case where `commandline -cp` returns a line ending with a space.

### 3. Fish Doesn't Properly Erase Completions (fish-shell#6716)

The `complete -e 'cmd'` in the snippet is meant to erase existing completions. Known issue: `complete -e` may not properly erase completions in all cases. The workaround used by carapace-bin is to create an **empty stub file** at `~/.config/fish/completions/{cmd}.fish`.

### 4. Special Characters in Values

Fish interprets certain characters as special syntax. Carapace's fish formatter does **not** perform any shell-specific escaping of special characters in completion values — it relies on fish's built-in escaping when displaying candidates.

### 5. No Per-Candidate Nospace

Fish's tab-separated format has no way to communicate per-candidate nospace information. Directory completions ending in `/`, colon-separated values like `origin/main`, and comma-separated values all get an unwanted trailing space.

### 6. No Style/Color Support

Fish's `value\tdescription` format has no mechanism for per-candidate colors.

### 7. `IFS=''` in the Snippet

The `IFS=''` before `set data` is critical. Without it, fish would split the completion output on whitespace, destroying values that contain spaces.

### 8. Integration Tests Are Commented Out

In `example/main_test.go`, the fish integration tests are entirely commented out. Fish completions are only tested via the golden file snippet test and unit tests.

## Completion Dispatch Flow for Fish

```
User presses TAB
  → Fish invokes _cmd_completion function
  → Snippet: commandline -cp gets command line up to cursor
  → Snippet: 3-stage open-quote retry with xargs
  → carapace _carapace fish subcommand receives args
  → complete() in complete.go detects shell
  → NO fish-specific patching (no case "fish" in switch)
  → traverse(cmd, args[2:]) → resolve action + context
  → action.Invoke(context) → InvokedAction
  → InvokedAction.value("fish", currentWord)
  → shell.Value() applies global preprocessing:
      1. Color disable if NO_COLOR
      2. Prefix filtering (unless CARAPACE_UNFILTERED)
      3. Message integration (ERR values for fish)
      4. Nospace propagation (dead code for fish — ignored by formatter)
      5. Sort + dedup + clear UIDs
  → fish.ActionRawValues(currentWord, meta, values):
      1. Sanitize values (strip \n, \r, \t)
      2. Format as "value\tdescription" per line
      3. Join with \n
  → Snippet: echo $data
  → Fish parses tab-separated output, displays in pager
```
