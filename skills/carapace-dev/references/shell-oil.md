## Oil Background

Oil (OSH) is a Bash-compatible shell that uses the same `COMP_LINE`/`COMP_POINT` mechanism as bash for programmable completion. However, it lacks some of the advanced features of bash completion (no `COMP_TYPE` successive-tab display, no `COMP_WORDBREAKS` handling, and no redirect patching in carapace).

## The Oil Snippet

Generated by `oil.Snippet(cmd)` in `internal/shell/oil/snippet.go`:

```bash
#!/bin/osh
_example_completion() {
  local compline="${COMP_LINE:0:${COMP_POINT}}"
  local IFS=$'\n'
  mapfile -t COMPREPLY < <(echo "$compline" | sed -e "s/ $/ ''/" -e 's/"/\"/g' | xargs example _carapace oil)
  [[ "${COMPREPLY[*]}" == "" ]] && COMPREPLY=() # fix for mapfile creating a non-empty array from empty command output
  [[ ${COMPREPLY[0]} == *[/=@:.,$'\001'] ]] && compopt -o nospace
  # TODO use mapfile
  # shellcheck disable=SC2206
  [[ ${#COMPREPLY[@]} -eq 1 ]] && COMPREPLY=(${COMPREPLY[@]%%$'\001'})
}

complete -F _example_completion example
```

### Key Differences from Bash

1. **Shebang**: `#!/bin/osh` instead of no shebang
2. **Simpler retry**: Uses `sed -e "s/ $/ ''/" -e 's/"/\"/g'` instead of the 3-stage xargs retry
3. **Nospace**: Hardcoded pattern `[/=@:.,$'\001']` instead of the boolean `true/false` flag
4. **No COMP_TYPE/COMP_WORDBREAKS**: Doesn't export these variables
5. **Single-entry nospace removal**: `COMPREPLY=(${COMPREPLY[@]%%$'\001'})` strips the `\001` indicator from all values

## Value Formatting: `ActionRawValues()`

`oil.ActionRawValues(currentWord, meta, values)` in `internal/shell/oil/action.go` is a simpler formatter than bash (~37 lines):

### Output Format

- Single candidate: just the value (with `\001` appended if nospace)
- Multiple candidates: `value (description)` format with `\001` appended to nospace values

### Nospace Indicator

The `\001` (SOH) character is appended to values where nospace matches:

```go
if meta.Nospace.Matches(val.Value) {
    val.Value = val.Value + nospaceIndicator
}
```

The snippet checks for `\001` in the completed value: `[[ ${COMPREPLY[0]} == *[/=@:.,$'\001'] ]] && compopt -o nospace`

Then strips it: `COMPREPLY=(${COMPREPLY[@]%%$'\001'})`

### Sanitizer

Strips `\n` and `\t` (but not `\r` unlike bash).

## Completion Dispatch Flow for Oil

```
User presses TAB
  → Oil invokes _cmd_completion()
  → Snippet constructs compline and calls: echo "$compline" | sed | xargs cmd _carapace oil
  → carapace _carapace oil subcommand receives args
  → complete() in complete.go detects shell=oil
  → NO oil-specific patching (TODO in complete.go: "what about oil and such?")
  → traverse(cmd, args[2:]) → resolve action + context
  → action.Invoke(context) → InvokedAction
  → InvokedAction.value("oil", currentWord)
  → shell.Value() applies global preprocessing
  → oil.ActionRawValues(currentWord, meta, values):
      1. Append \001 to values where nospace matches
      2. Format single value or value (description)
      3. Return newline-separated values
  → Snippet parses output: mapfile -t COMPREPLY
  → Check nospace pattern and strip \001
  → Oil displays completions
```

## Edge Cases

### No Redirect Patching

The TODO in `complete.go` notes:

```go
case "bash": // TODO what about oil and such?
```

Oil uses the same `COMP_LINE`/`COMP_POINT` mechanism as bash but doesn't receive redirect patching or COMP_WORDBREAKS handling. If oil suffers from the same redirect leaking problem as bash, completions may include redirect tokens as positional arguments.

### Hardcoded Nospace Pattern

The nospace check in the snippet uses a hardcoded pattern `[/=@:.,$'\001']` rather than the dynamic `SuffixMatcher` approach. This means nospace only works for values ending in `/`, `=`, `@`, `:`, `.`, or `,` — other nospace suffixes are not recognized.
